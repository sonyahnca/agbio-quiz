# 시험 대비 퀴즈 PWA

위키 drills를 문제은행으로 쓰는 정적 웹앱. 빌드·서버·비용 없음.

## 기능
- 🎲 **랜덤 퀴즈**: 장별 필터 + 보기 셔플 + 즉시 채점·해설 (서술형 포함 옵션)
- 📝 **모의고사**: 무작위 N문항(객관식) + 제한시간 타이머 + OMR 답안지 + 제출 후 일괄 채점·장별 정답률·오답 리뷰
- 🔁 **약점 복습**: 최근 틀린 문항만 다시
- 📕 **오답노트**: 누적 오답을 **풀이와 함께 영구 저장** + **장별 오답 개수 분포(+정리노트 슬러그)**로 보충학습 우선순위 / '이해함' 제거 / 현재약점·교정됨 태그
- 📊 **통계/진도**: 장별 누적 정답률(localStorage, 약점 장 표시)

## 실행 방법
**1) 그냥 열기 (가장 간단)**
- `index.html`을 브라우저로 더블클릭. (데이터를 .js로 임베드해 `file://`에서도 작동)

**2) 로컬 서버 (오프라인 PWA 설치까지)**
```
node _app/serve.js     # → http://localhost:8778
```

**3) 태블릿에서 쓰기 (추천)**
- GitHub Pages 등 https에 `_app/` 전체를 올림 → 태블릿 브라우저로 접속
- Safari/Chrome "홈 화면에 추가" → 오프라인 설치 앱처럼 사용(서비스워커 캐시)

## 문제 데이터 갱신
drills(`농업생물화학/wiki/drills/*.md`)를 고치면 다시 생성:
```
python3 _app/build_questions.py     # → data/농업생물화학.json + .js 재생성
```
(서비스워커는 network-first(quiz-v2)라 온라인이면 갱신이 바로 반영됨)
(파서는 `**N. 질문**` → `①②③④` 보기 → `<details>정답·해설</details>` 형식을 읽는다.)

## 과목 추가
1. 그 과목 drills를 같은 형식으로 만들고 파서로 `data/<과목>.js` 생성
2. `index.html`에 `<script src="data/<과목>.js"></script>` 한 줄 추가
3. 앱 홈 상단에 과목 선택 탭이 자동 생성됨
